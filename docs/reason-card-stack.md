# 02「選ばれる理由」スマホ版カードスタック演出 仕様書

対象: `givecareer-white-lp/index.html` のスマホ表示（720px以下）における
理由カード3枚（`#reason-1` `#reason-2` `#reason-3`）の「スクロールで重なっていく」演出。

最終更新: 2026-07-23

---

## 1. 仕組みの全体像

3つの部品で成立している。**すべて `index.html` 内**にある。

| 部品 | 場所 | 役割 |
|---|---|---|
| CSS（sticky設定） | `@media (max-width: 720px)` 内の `.reasons-list` / `.reason` まわり | カードを画面上部に固定し、24pxずつずらして重ねる |
| JS `syncReasonStackLayout` | `/* スマホ：理由カードは自然な高さのまま… */` ブロック | 各カードの `margin-bottom` を実際の高さから自動計算 |
| JS `updateReasonStack` | 同上 | 下に重なったカードからはみ出す部分を `clip-path` でカット |

### 動きの流れ

1. 01が画面上部28pxに固定される（チラ見せなし）
2. スクロールすると02が下から来て01に重なる。02は52px位置に固定 → 01の上端24pxがチラ見え
3. 03も同様に76px位置へ → 01・02が24pxずつチラ見え（3枚スタック完成）
4. さらにスクロールすると**3枚が一体となって**上へ退場していく

### なぜJSでmarginを計算するのか（重要）

カードの高さは文章量・画面幅で変わる。高さがバラバラのまま重ねると、
背の高いカードの下端が短いカードの下からはみ出す。これを `clip-path` で隠しているが、
**クリップの更新はscrollイベント（メインスレッド）でしか動かない**。
一方、stickyカードの位置はブラウザのコンポジタが毎フレーム動かす。

そのため「カード同士が相対移動する時間」が長いと、クリップが1フレーム遅れて
**下端がチラチラ見えるバグ**になる。これを防ぐ条件が:

> **不変条件: どのカードも `margin-bottom + 高さ` が、末尾カードから24px刻みで揃っていること**
> （margin1+h1 = margin3+h3+48、margin2+h2 = margin3+h3+24）

この条件が成立すると、3枚が**同時に**退場を始めて一体で動くので、
相対移動が起きずクリップも一定値のまま＝チラつかない。
`syncReasonStackLayout` はこの条件を満たすmarginを毎回自動計算している。

## 2. いじりたいとき早見表

### ✅ 登場テンポ・余韻を変えたい（02/03を早く/遅く出す）

JS内の **`reasonEndMargin`（現在 92）** を1つだけ変える。

```js
var reasonEndMargin = 92;
```

- **意味**: 03が固定されてから退場が始まるまでのスクロール量(px)。
  全カードのmarginがこれに連動するので、大きくすると全体にゆったり、小さくすると詰まる
- 旧仕様のゆったりテンポに戻すなら `226`（=以前の274/250/226時代と同じ）
- ついでにCSSフォールバック値（下記 §3）も合わせておくとJS無効環境でも揃う

**⚠️ CSSやJSで個別カードのmargin-bottomを直接いじらないこと。**
不変条件が崩れて退場時のチラつきバグが再発する（過去2回これで壊れた）。

### ✅ チラ見せの幅（24px）を変えたい

2箇所をセットで変える:

1. JS: `var reasonPeek = 24;`
2. CSS: 固定位置のずらし幅
   ```css
   .reason:nth-of-type(2) { top: calc(var(--reason-stack-top) + 24px); }
   .reasons-list #reason-3 { top: calc(var(--reason-stack-top) + 48px); } /* ← peek×2 */
   ```

### ✅ スタックの固定位置（上端からの距離）を変えたい

CSS変数 `--reason-stack-top`（`.reasons-list` に定義、現在 `max(28px, env(safe-area-inset-top))`）を変える。JS修正は不要。

### ✅ カードの文章量・中身を変えたい

**何もしなくていい。** 高さはJSが実測して全部自動調整する（フォント読込後・リサイズ時も再計算）。

### ✅ カードの角丸を変えたい

`.reason { border-radius: 18px }` を変えたら、JS `updateReasonStack` 内の
安全マージン **48**（`rects[j].bottom - 48`）が「角丸+30px以上」あるか確認。
この48は「角丸の丸みからの覗き防止＋高速スクロール時のクリップ1フレーム遅れ対策」の余裕。

### ✅ カードを増やす／減らす

1. HTML: `.reasons-list` 内に `article.reason` を追加（idは `reason-4` 形式）
2. JS: `reasonCards` 配列にそのidを追加
3. CSS: 固定位置とz-indexを追加
   ```css
   .reasons-list #reason-4 { top: calc(var(--reason-stack-top) + 72px); z-index: 4; margin-bottom: 92px; }
   ```
   ※ topは24px刻みで増やす。末尾カードのCSSフォールバックmarginは `reasonEndMargin` と同値に
4. margin計算・クリップは自動で追従する

### ✅ デスクトップ側を変えたい

この仕組みは `innerWidth > 720` では**全て解除**される（inline style をクリア）。
デスクトップは通常のgrid縦積み＋sticky目次レール。自由に変えてよい。

## 3. 現在の設定値まとめ

| 項目 | 値 | 場所 |
|---|---|---|
| ブレークポイント | 720px以下で発動 | CSS media query / JS `innerWidth > 720` |
| 固定位置 | 28px（+safe-area） | CSS `--reason-stack-top` |
| チラ見せ幅 | 24px | JS `reasonPeek` ＋ CSS top のずらし |
| 退場前の余韻 | 92px | JS `reasonEndMargin` |
| クリップ安全マージン | 48px | JS `updateReasonStack` 内 |
| カード角丸 | 18px | CSS `.reason` |
| CSSフォールバックmargin | 140 / 116 / 92px | `.reason:nth-of-type(1/2)` / `#reason-3` |
| リスト末尾の負マージン | -92px（JSが上書き） | CSS `.reasons-list` |

※ CSSのmargin-bottom値（140/116/92）とリストの-92pxは**JS無効時のフォールバック**。
JS有効時は `syncReasonStackLayout` が実高さに合わせた値でinline上書きする
（例: 390px幅で 66/24/148＋リスト-148）。marginが24px未満になりそうな場合は
末尾marginを自動で持ち上げて不変条件を守る。

## 4. 過去にやらかしたバグ集（再発防止）

| 症状 | 原因 | 教訓 |
|---|---|---|
| カードに謎の大きい余白 | 旧JSが3枚の高さをmin-heightで最大値に強制統一していた | 高さ統一はやめて自然高さ＋クリップ方式に |
| 02が03の下端から数px覗く | クリップを「重なるカードの下端ぴったり」で切っていて、角丸18pxの丸みの隙間から見えた | 安全マージンを角丸より深く取る（現在48px） |
| スクロール中に下端がチラチラ見える | 個別カードのmarginを手編集して不変条件（24px刻み）が崩れ、退場時にカードが別々に動いた。クリップは1フレーム遅れる | **marginは手でいじらない。テンポは `reasonEndMargin` だけで調整** |

## 5. 動作確認のしかた

スマホ幅で以下の4状態を見る（Chromeレスポンシブモード or 実機）:

1. **02が01に重なる途中** — 01の下端が02の下から見えないか
2. **3枚スタック完成時** — 01・02のチラ見せが24pxずつ揃っているか
3. **退場中**（CTAが見えてくるあたり）— 3枚の下端が揃って動くか、下から白い端が覗かないか
4. **速いフリックで往復** — チラつきがないか

文章量を変えたあとは念のため390px幅と500px幅の両方で見る（高さバランスが変わりmarginの自動補正が効くため）。
